## 12 Sections

### A-01: Architecture Patterns
Diagrams of: Monolith, Modular Monolith, Microservices, Serverless, Event-Driven. Decision tree. Modular monolith as new default. Hexagonal/Clean as internal structure. DDD for boundaries.

### A-02: Scale Tiers
4-tier matrix: Solo (1 dev) → Small Team (2-10) → Growth (10-50) → Enterprise (100+). Architecture, deployment, database, key investment for each.

### A-03: Modular Design Principles
Principle boxes for: Module boundaries, Dependency Inversion, Plugin architectures, Event-driven decoupling. Tools: Spring Modulith, .NET Aspire, Packwerk, Service Weaver.

### A-04: Frontend Landscape
Comparison table: Next.js, Nuxt, SvelteKit, Remix/RR7, Astro. Tailwind+shadcn/ui as default. HTMX for hypermedia renaissance.

### A-05: Backend Landscape
Comparison table: Node/Bun, Go, Rust, Python/FastAPI, Rails, .NET, Java/Spring, Elixir/Phoenix.

### A-06: API & Communication
REST vs GraphQL vs gRPC vs tRPC comparison. Event-driven patterns. Schema-first vs code-first.

### A-07: Data Architecture
Database selection matrix. Postgres dominance. NoSQL lanes. NewSQL/Distributed SQL. Vector DBs. Edge DBs.

### A-08: Infrastructure & Deployment
K8s (82%), K3s, PaaS (Railway/Vercel/Fly.io), Serverless/Wasm, IaC (Terraform/OpenTofu/Pulumi), CI/CD, Platform Engineering (Backstage 89%), GitOps.

### A-09: Observability & Reliability
Three pillars, OpenTelemetry as standard, LGTM stack, SRE practices (SLOs, error budgets, DORA).

### A-10: Security Architecture
Auth (OAuth2, Passkeys/WebAuthn, Session vs JWT), Zero-trust (Istio, Linkerd), API Gateways, Secrets (Vault, SOPS), Supply chain (SBOMs, Sigstore, SLSA).

### A-11: AI Context Architecture
"Context as Code" pattern. Tool comparison (CLAUDE.md, copilot-instructions.md, .cursorrules, AGENTS.md). Context engineering principles. MCP (97M+ downloads). Project structure for AI legibility.

### A-12: Reference Stacks
Concrete picks per tier: Solo, Small Team, Growth, Enterprise, Content Sites, ML/AI, Real-Time, Internal Tools. User's personal stack (React/Tailwind/shadcn/Next.js/Railway) and work stack (Ruby/Oracle/K8s) as real-world references.
